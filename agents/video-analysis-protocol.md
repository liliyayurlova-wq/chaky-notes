# Video Analysis Protocol (Версия 1.0)

**Цель:** Системный анализ найденных YouTube-видео для извлечения практических инсайтов, которые мы сразу применяем в продукте (контент-план, рекомендации каналов, Pinterest-визуал).

**Триггер запуска:** После поиска YouTube API (External Promotion Monitor или ad-hoc запрос).

---

## Шаг 1. Первичный отбор

❗Пропускаем видео, если:
- Длительность > 40 минут (если это не детальный кейс)
- Канал — мусор/накрутка подписчиков
- Тема не связана с нашей (селлеры, внешнее продвижение, контент, Pinterest, TG)

Оставляем, если:
- Связано с любой из групп нашего мониторинга
- Короткое (до 20 мин) — в приоритете
- Есть практические цифры, примеры, промпты, скрипты

---

## Шаг 2. Скачивание транскрипта

```
yt-dlp --write-auto-sub --sub-lang ru --skip-download -o "tmp/transcripts/%(id)s" <URL>
```

Формат: `.vtt` → конвертируем в чистый текст (убираем тайминги, мусор).

Если субтитров нет → пропускаем видео (нечего анализировать).

---

## Шаг 3. ИИ-анализ содержания

Модель: DeepSeek через OpenRouter (или другая доступная)

**Промт для анализа:**

```
Ты — аналитик по внешнему продвижению для селлеров Wildberries/Ozon. 
Проанализируй транскрипт видео. Выдай структурированный результат:

1. **Тема и суть** (2-3 предложения)
2. **Ключевые инсайты** (список, 3-5 пунктов)
3. **Практические цифры** (бюджеты, конверсии, сроки — если есть)
4. **Конкретные инструменты/сервисы** (названия, ссылки)
5. **Применимость к нашему продукту:**
   - Можно применить в контент-плане? Да/Нет
   - Можно применить в Pinterest? Да/Нет
   - Можно применить для рекомендаций каналов? Да/Нет
   - Оценка полезности: 1-10
6. **Что можно автоматизировать** (из того, что показано в видео)
```

---

## Шаг 4. Сохранение результата

- Файл: `obsidian/clients/_research/video-analysis-YYYY-MM-DD-ID.md`
- Формат: Markdown с YAML frontmatter
- В frontmatter: `video_id`, `channel`, `title`, `url`, `practical_value` (1-10), `topics` (список тегов)
- В теле: результат анализа (из Шага 3)

---

## Шаг 5. Извлечение action items

После каждого анализа отвечаем на вопросы:
1. **Что из этого мы можем сделать прямо сейчас?** (technology, prompt, script)
2. **Как это повлияет на структуру нашего продукта?** (новый блок в контент-плане? новый тип рекомендаций?)
3. **Стоит ли сделать пост в @delay_trafik на эту тему?**

Action items записываются в:
- `memory/YYYY-MM-DD.md` — если это инсайт для продукта
- `content/posts/` — если пост в канал

---

## Шаг 6. Раз в неделю: синтез

Собираем все action items за неделю → формируем:
1. Список изменений в продукте (что добавить/убрать)
2. Идеи для постов в @delay_trafik
3. Архив: если видео полезное → переносим ссылку в `video-cases-YYYY-MM-DD.md`

---

## Частота

| Активность | Когда |
|---|---|
| Поиск видео (External Monitor) | Раз в 2 дня |
| Анализ новых видео | После каждого поиска (если найдено >0) |
| Раз в неделю синтез | Воскресенье |

---

## Критерии «стоп»

Анализ не делается, если:
1. Найдено 0 новых видео (относительно предыдущего поиска)
2. Subbot занят другим анализом (очередь)
3. Видео без субтитров
4. Канал-помойка (посты без содержания)
